# Windows full-repo bug audit — 2026-06-03

Exhaustive, adversarially-verified audit of the Windows app (Rust engine + WinUI 3 / .NET 8)
driven by four multi-agent workflows, then a fix pass on a branch
(`win-prod-hardening-2026-06-03`). Trigger: user-reported screen flicker + "engine problems",
plus a "leave no line unturned" / max-performance mandate.

## Method

Four background workflows, each **find → adversarially verify (refute-by-default)**:

| WF | Scope | Raw → Confirmed |
|----|-------|-----------------|
| 1 | Rust engine, 15 subsystem cells, line-by-line | 27 → 19 |
| 2 | WinUI app + flicker/DispatcherObject, 12 cells | 54 → 29 |
| 3 | Cross-file flow: scan pipeline, IPC contract, DB/epoch, cancellation, silent-failure | 24 → 22 |
| 4 | Performance: hot-loop allocs, copies, locks, EP/GPU, query | 38 → 8 |

**78 confirmed** findings after dedup ≈ 70 distinct. Verifiers rejected ~40 false positives with
real rigor (compiled the vendored tokio source to disprove a "deadlock"; ran standalone rusqlite
`EXPLAIN QUERY PLAN`; reproduced the WinVerifyTrust leak on the live box; proved WAL sidecars are
cleaned at connection-close). Baseline before fixes: engine clippy clean + 255 tests; app build
0/0 + tests + format clean.

The user's **flicker** complaint is fully explained: search-box keystrokes wrote XAML off the UI
thread; Deep Analyze's stale `Complete` fought the live UI at ~4 Hz on every 2nd+ run; the pipeline
strip blanked to grey on every scan completion; Cleanup/People rebuilt entire lists (Clear+Add)
~1 Hz during scans.

## FIXED this session (verified: engine clippy `-D warnings` + 256 tests; app build + tests + format)

### Engine — HIGH (data-loss / crash / deadlock)
1. **`commands/face_clustering.rs`** — re-clustering ran `DELETE FROM persons` then re-inserted with
   `name=NULL`, **wiping every user-assigned name + "not this person" verdict on every scan
   completion** (auto-fired). Fix: snapshot prior identities, re-attach each new cluster's identity
   from the prior person owning the majority of its member faces (tie → anchor face).
2. **`pipeline/tagging.rs` + `pipeline/dbwriter.rs`** — a per-file 60 s timeout row or a GPU-dead
   short-circuit emits empty tags, and the dbwriter `DELETE FROM tags … source='auto'` ran
   **unconditionally** (unlike the faces/OCR/doc stage-ran gates), wiping a file's prior auto-tags
   on a transient slow read or mid-scan GPU TDR. Fix: added `tags_evaluated` flag + gate.
3. **`pipeline/restructure_apply.rs`** — `move_file`/`make_symlink`/`unique_destination` omitted the
   `\\?\` long-path prefix, so restructure moves into deep semantic folders silently failed past
   MAX_PATH (260). Fix: wrap operands in `to_extended_length` (matches every other FS site).
4. **`models/vlm.rs`** — the per-file Deep Analyze CLI piped stderr but never drained it → child
   **deadlocks** when the ~64 KB stderr pipe fills (no outer timeout). Fix: `Stdio::null()` for
   stderr (matches `vlm_server.rs`).
5. **`models/runtime.rs`** — a `cpu` EP override was silently ignored (the vendor GPU EP still
   bound), defeating the documented GPU-TDR recovery path → crash-loop. Fix: CPU override now
   returns a CPU-exclusive chain.
6. **`pipeline/dbwriter.rs` (`heal_candidate_moved`)** — a `file_ref` (NTFS MFT id) is volume-local,
   so two distinct files on different volumes (or two hardlinks) could collide and **collapse into
   one catalog row** (FK-cascading the loser's tags/faces). Fix: require old-path-gone for ALL heals
   (the `by_ref` unconditional short-circuit is gone); coexisting files stay distinct. Tests updated.
7. **`commands/embed.rs`** — the CLIP-text encoder binds a GPU EP **outside** the scan's `ep_guard`
   arm window (first search query), so a bad-pack crash left no breadcrumb → search crash-loop. Fix:
   wrap the lazy `ClipText::load` in `ep_guard::arm/disarm`.
8. **`commands/wipe.rs` + `main.rs`** — wipe-during-scan interleaved truncate with the live DbWriter,
   leaving a half-populated "wiped" library. Fix: engine cancels any in-flight scan and bounded-waits
   for the writer to release before truncating (engine is the sole DB owner).

### Engine — MEDIUM
9. **`downloader.rs`** — a range resume that got `200` (full body) instead of `206` appended the full
   file after the partial prefix → corrupt part. Fix: require 206 on resume, else discard + restart.
10. **`downloader.rs`** — corrupt `.part-NN` files weren't cleaned on SHA/size failure, so a
    byte-complete-but-wrong part was treated as "already complete" on retry → permanently stuck
    install. Fix: remove parts on SHA/size mismatch.
11. **`pipeline/dbwriter.rs`** — rename-heal `LIMIT 1` with no `ORDER BY` could return a still-present
    copy ahead of the genuinely-moved row → orphaned the moved file's row. Fix: order file_ref
    matches first, iterate, heal the first whose old path is gone.

### Engine — perf / low (WF4 + WF1)
12. **`ipc/bounded_read.rs`** — `mem::take(buf)` discarded the pre-grown 8 KB reuse buffer every
    frame. Fix: build from a borrow.
13. **`models/ram_plus.rs`** — `select_tags` allocated a lowercase `String` for ~4573 tags/image even
    with an empty suppress set. Fix: short-circuit the empty case.
14. **`shell/heic.rs`** — HEIC decode cap was 160 MP vs the pipeline's 50 MP per-worker RAM budget.
    Fix: 50 MP + corrected comment.
15. **`models/ep_guard.rs`** — `reenable()` cleared ALL crash-disabled EPs; verifying the CUDA pack
    re-armed a separately-poisoned OpenVINO. Fix: added per-EP `reenable_ep(ep)`; callers target one.
16. **`pipeline/dbwriter.rs`** — cancel dropped the in-flight batch buffer, discarding already-tagged
    files. Fix: flush the remaining buffer before breaking on cancel.
17. **`util/hnsw_index.rs`** — `search_top_k` allocated a fresh `Search` scratch per query, so
    `instant-distance` re-zeroed an n-byte visited set every call — an **O(n²)** term over a
    large-library clustering pass. Fix: reusable `Searcher` threaded through both kNN sweeps.
18. **`pipeline/face_clustering.rs` + `restructure_semantic.rs`** — brute-force kNN fully sorted all
    n−1 neighbors to keep k. Fix: `select_nth_unstable_by(k)` then sort the k.

### App — HIGH
19. **`ViewModels/LibraryViewModel.cs`** — the search debounce set `IsLoading`/`ErrorMessage` on a
    thread-pool thread, so x:Bind wrote `ProgressRing.IsActive`/`StatusText` **off the UI thread** (a
    native fast-fail). Fix: `OnUi` marshaling helper around all loading-state writes.
20. **`ViewModels/EngineClient.cs`** — `DeepAnalyzeComplete` was never cleared on a new run, so on the
    2nd+ Deep Analyze the stale "Done" state **clobbered the live UI at ~4 Hz** (buttons + text
    flickering). Fix: clear it on `DeepAnalyzeStarting` (and on scan start, #23).
21. **`Services/ModelInstallerService.cs`** — `Reset()` (engine-respawn hook) omitted RamPlus +
    Accelerator, so a crash mid-download left them **spinning forever** with no Retry (and a stuck
    RamPlus blocked "Install all" + onboarding). Fix: fail those slots too.

### App — MEDIUM
22. **`Views/Sidebar/SidebarPipelineProgress.xaml.cs`** — the pipeline strip **blanked to all-grey**
    for a beat on every scan completion (Completed → −1, latches not yet set). Fix: hold at the
    People stage on Completed (monotonic, never regresses to grey).
23. **`Views/Sidebar/SidebarPipelineProgress` + `EngineClient.Commands.cs`** — stale Deep-Analyze
    latches jumped a fresh rescan straight to "Done". Fix: clear the latches in `StartScanAsync`.
24. **`Services/ReadStore.cs`** — `RecentAsync` (default grid) omitted `failed = 0`, leaking
    corrupt/unreadable files as broken placeholder tiles. Fix: `failed = 0` base predicate (matches
    the macOS reference + sibling queries).
25. **`Services/WinVerifyTrustChecker.cs`** — the WVT state handle leaked on every engine spawn (the
    CLOSE re-marshaled a stale managed struct, zeroing `hWVTStateData`). Fix: read the native struct
    back before flipping to CLOSE (verifier reproduced + confirmed the fix on the live box).
26. **`Views/Library/FilePreviewSheet.xaml.cs`** — "Apply rename" declared success on the IPC send and
    collapsed the card even when the engine reported failure (silent-failure). Fix: await the
    `BulkActionResult`, keep the card open on `Failed > 0`.

### App — perf / low
27. **`SidebarPipelineProgress.xaml.cs`** — filled dots kept the 1.5 stroke left over from the active
    state. Fix: reset to 1 in the filled branch.
28. **`SidebarPipelineProgress.xaml.cs`** — `SyncStage` rewrote all 5 cells + allocated AutomationName
    strings on every 10 Hz `LastProgress` even when the stage hadn't changed. Fix: cache the last
    rendered index, early-return when unchanged.

## DEFERRED — verified-real, documented with the fix (next session)

Grouped by why each is deferred. Each is a confirmed finding from a workflow above.

### Needs on-hardware verification (project rule: no unverified pipeline/EP regression)
- **`models/runtime.rs::active_pack_dir`** (medium, crash) — `ORT_DYLIB_PATH` is pinned from the raw
  GPU vendor, ignoring a `gpuExecutionProviderOverride`, so a forced cross-vendor EP binds a provider
  DLL against a mismatched ORT base → crash the ep_guard can't recover. Fix: derive the pack from the
  override-aware `armed_provider()`. Defer: needs a real cross-vendor box (both packs installed) to
  confirm the bind/fallback.
- **`commands/bulk.rs` rename TOCTOU** (low, data-loss) — `std::fs::rename` is clobbering
  `MoveFileExW(REPLACE_EXISTING)`; an external file materializing in the existence-check→rename
  window is silently overwritten. Fix: no-clobber `MoveFileExW` (no `REPLACE_EXISTING`). Defer:
  Windows-specific overwrite semantics, on-hardware verify.

### Needs IPC-contract change (Rust + C# + schema lockstep)
- **`commands/prewarm.rs` + `main.rs`** (medium, concurrency) — one global `prewarm_cancel`
  AtomicBool: a per-row Cancel aborts ALL concurrent downloads, and a new prewarm un-cancels a
  pending cancel. Fix: per-`model_kind` cancel token; `cancelPrewarm` needs a `modelKind` field.
- **Schema drift** (low) — `FileDoneEvent.skippedStages` (Rust-only), `DeepAnalyzeProgress.currentCaption`
  + the `HardwareInfo` field set (Rust/C# only) are absent/under-specified in `ipc.schema.json`.
  Fix: promote them to the canonical schema + mirror Swift.

### Cross-platform DB lockstep (needs a coordinated migration / a Mac)
- **`Services/ReadStore.cs` kind-filtered grid** (small, perf) — `WHERE kind=? ORDER BY scanned_at`
  forces a TEMP B-TREE sort (~55 ms/refresh at 50 K with a saved filter). Fix: composite index
  `(kind, scanned_at)` — but that is a new `vN+1` migration that must land on **both** platforms to
  preserve GRDB byte-faithful lockstep.
- **`pipeline/dbwriter.rs` `created_at`** (low, parity) — always NULL on Windows-written libraries
  (macOS populates it). Fix: collect `metadata.created()` in discovery, thread through, bind it.

### App — flicker / silent-failure (next, no new infra needed)
- **`ViewModels/CleanupViewModel.cs` + `PeopleViewModel.cs`** (medium, flicker) — `Replace()` does
  `Clear()+Add` (CollectionChanged.Reset), re-realizing the whole ItemsRepeater ~1 Hz during a scan,
  re-decoding every anchor BitmapImage, and **discarding in-flight keeper/selection state**. Fix:
  port `LibraryViewModel.MergeById` identity-stable merge; add `DecodePixelWidth` to AnchorImage.
- **`Views/People/PeopleView.xaml.cs`** (medium) — select-mode breaks after a refresh rebuilds
  clusters (new cards have no IsSelected subscription, checkboxes vanish, selection wiped). Fixed
  for free by the identity-stable merge above, or re-wire on refresh.
- **`Views/Library/LibraryView.xaml.cs:1057`** (HIGH, silent-failure) — trash is fire-and-forget and
  **removes tiles even for files the engine couldn't trash** (open handle / permission), so the user
  believes files were recycled when they're still on disk. Fix: await the `trashFiles`
  `BulkActionResult`, remove only `Ok` tiles, refresh on `Failed>0`. Defer: the optimistic-removal
  path coexists with `UndoStack.CaptureNextBulkResult` (double result-listener) — needs the WinUI
  runtime to verify the trash+undo interaction; self-heals on the next refresh meanwhile.
- **`Views/Library/FilePreviewSheet.xaml.cs:786` (Apply tags)** + **`Views/People/PersonDetailSheet.xaml.cs:117`
  (rename)** (medium/low, silent-failure) — same fire-and-forget class as the rename fix already
  applied; route through `WaitForBulkActionResultAsync`. (PersonDetail also runs synchronous SQLite +
  up to 200 `File.Exists` on the UI thread — move to `Task.Run`.)
- **`Views/Restructure/TreeDiffControl.cs:152`** (medium) — the Tree view's TreeViews have no
  ItemTemplate, so every node renders the literal type name. Fix: add an ItemTemplate binding
  `DisplayName`/`HighlightBrush`.
- **`Views/Restructure/SankeyFlowControl.cs:96`** (medium, perf/jank) — `Render()` runs an
  O(288 × Moves) barycentric scan on the UI thread on every `SizeChanged` with no debounce. Fix:
  precompute a flow matrix once + debounce resize. (`:460` — ribbon tap is dead on touch/pen; hit-test
  at the tap point.)

### App — low / cosmetic / micro-perf
- `SankeyFlowControl` touch-tap; `ThumbnailDiskCache.cs:211` sweep-throttle cap overshoot;
  `ModelInstallerService.cs:622` watchdog token always default; `ShimmerView.xaml.cs:26`

### Engine — micro-perf
