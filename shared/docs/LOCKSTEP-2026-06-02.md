# Windows ↔ macOS lockstep reconciliation — 2026-06-02

Source: adversarial parity-audit workflow (56 agents, 39 confirmed of 48 candidates). macOS is the visual+behavioral reference; the cross-platform DB round-trip is the hard invariant.

## CRITICAL (2)

### [tokens-0] Timestamp epoch mismatch: Windows writes files.scanned_at/modified_at/created_at as UNIX epoch; macOS writes them as Apple reference-date (2001) epoch — a 978307200s (~31yr) offset
- **class**: breaks-round-trip  |  **edit**: macos  |  **win_verifiable**: False  |  **severity**: critical
- **Windows**: platforms/windows/src/engine/src/pipeline/tagging.rs:1114-1117 scanned_unix = SystemTime::now().duration_since(UNIX_EPOCH).as_secs_f64(); discovery.rs:280 modified = mtime.duration_since(UNIX_EPOCH); dbwriter.rs:270-296 binds f.modified_unix (?5)/f.scanned_unix (?6) into modified_at/scanned_at — all seconds since 1970.
- **macOS**: platforms/apple/engine/Sources/FileIDEngine/Storage/DBWriter.swift:331-333 binds file.createdAt?.timeIntervalSinceReferenceDate, file.modifiedAt?.timeIntervalSinceReferenceDate, Date().timeIntervalSinceReferenceDate into created_at/modified_at/scanned_at — seconds since 2001-01-01. Restructure.swift:133-135 self-documents this: "created_at/modified_at are seconds since the 2001 reference date; shift to the Unix epoch (+978_307_200) so day-of-year matches the Windows engine."
- **Impact**: A DB written on macOS, opened on Windows, reports every file's modified/scanned/created time ~31 years in the PAST (e.g. a 2026 photo reads as ~1995). Windows discovery.rs uses `scanned_at >= modified_unix` as a skip-rescan heuristic and date-derived Year_ tags would be wrong; the orphan sweep / restructure date-bucketing produce garbage years. Reverse direction (Windows DB on macOS) reads times ~31yr in the FUTURE. The column is a bare REAL with no epoch tag, so neither side can detect the mismatch — silent corruption.
- **Fix**: Make macOS write the same epoch Windows does: change DBWriter.swift:331-333 to file.createdAt?.timeIntervalSince1970, file.modifiedAt?.timeIntervalSince1970, Date().timeIntervalSince1970, and DeepAnalyzeRunner.swift:276 to Date().timeIntervalSince1970. Then macOS read sites that reconstruct Dates from these columns must use Date(timeIntervalSince1970:) instead of Date(timeIntervalSinceReferenceDate:) — Restructure.swift:135 (drop the +978_307_200 shift), Restructure.swift:161-162, ReadStore.swift:827 (vlm_analyzed_at). This makes the on-disk REAL canonically unix-epoch on both platforms (byte-faithful: same float for the same instant).

### [db-schema-3] Face embeddings are NOT aligned before embed on macOS — FaceAlign.align112 exists but has zero callsites; Windows aligns via face_align::align_112 before SFace
